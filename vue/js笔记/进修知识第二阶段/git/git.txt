==========================删除文件==============
# 删除暂存区和工作区的文件
$ git rm filename  
# 只删除暂存区的文件，不会删除工作区的文件
$ git rm --cached filename 
如果在配置 .gitignore 文件之前就把某个文件上传到远程仓库了，
这时候想把远程仓库中的该文件删除，此时你配置 .gitignore 文件也没有用，
因为该文件已经被追踪了，但又不想在本地删除该文件后再重新提交到远程仓库，
这时候可以使用 git rm --cached filename 命令取消该文件的追踪，
这样下次提交的时候，git 就不会再提交这个文件，从而远程仓库的该文件也会被删除

===========================新建分支
# 在别的分支下新建一个分支，新分支会复制当前分支的内容
# 注意：如果当前分支有修改，但是没有提交到仓库，此时修改的内容是不会被复制到新分支的
$ git branch branchname    新建分支
# 切换分支(切换分支时，本地工作区，仓库都会相应切换到对应分支的内容)
$ git checkout branchname    切换分支
# 创建一个 aaa 分支，并切换到该分支 （新建分支和切换分支的简写）
$ git checkout -b aaa 
# 可以看做是基于 master 分支创建一个 aaa 分支，并切换到该分支
$ git checkout -b aaa master

# 新建一条空分支（详情请看问题列表）
$ git checkout --orphan emptyBranchName
$ git  rm -rf .

$ git branch -D branchname    # 强制删除一个本地分支，即使包含未合并更改的分支

git reset --soft    commit_id          软性回滚, 跟 rebase 常规用法差不多的效果，把提交的东西丢回暂存区和工作区，
git reset --soft HEAD~1             软回滚一个版本,可以理解为撤销最近一次的 commit
git reset --mixed commit_sha1   清除暂存区但保留工作区变动。

git  rm --cached   list.html;           强制停止追踪 (工作区文件有改变)

      --soft   会保留工作  暂存区的内容也不会被清空   
      默认mixed保留工作目录，并清空暂存区,然后把commit中的记录放到暂存区

# 将本地仓库的文件推送到远程分支
# 如果远程仓库没有这个分支，会新建一个同名的远程分支

git push <远程主机名> <本地分支名>:<远程分支名>

git push -u origin master  建立当前分支和远程分支的追踪关系  有关系了 就可以git push

git push --all origin  不管是否存在对应的远程分支，将本地的所有分支都推送到远程主机

# 拉取所有远程分支到本地镜像仓库中
    git pull
    git pull origin2  master   
# 拉取并合并项目其他人员的一个分支 
   git pull origin branchname  
# 等同于 fetch + merge
   git fetch origin branchName
   git merge origin/branchName

如果远程主机的版本比本地版本更新，推送时 Git 会报错，要求先在本地做 git pull 合并差异，
